Aurora Platform Components (HLD Extract)
ComponentCategoryOwnerDescription
Argo CDPlatform ManagementSIG: ArchitectureArgo CD is Aurora’s primary GitOps engine for deploying and managing platform and application components across Kubernetes clusters. It continuously reconciles the desired state defined in Git repositories with the live state of clusters, providing drift detection, automated synchronization, and auditable change history. Argo CD enables consistent, repeatable deployments while enforcing separation of concerns between platform teams and application teams.

https://argo-cd.readthedocs.io/
cert-managerSecuritySIG: Securitycert-manager is used within Aurora to automate the issuance, renewal, and lifecycle management of TLS certificates for Kubernetes workloads and platform services. It integrates with external certificate authorities and internal PKI systems to ensure secure, policy-compliant encryption for ingress, service-to-service communication, and platform endpoints, reducing operational risk and manual certificate handling.

https://cert-manager.io/
CIDR AllocatorPlatform NetworkingSIG: NetworkingCIDR Allocator is a custom Aurora platform component responsible for allocating, tracking, and managing IP address ranges for Kubernetes clusters and networking domains. It ensures that cluster, pod, and service CIDRs are assigned deterministically and without overlap across environments. CIDR Allocator enables scalable multi-cluster networking, simplifies routing and firewall integration, and supports Aurora’s zero-trust and multi-cloud networking architecture.

https://github.com/gccloudone-aurora/cidr-allocator
CiliumNetworkingSIG: NetworkingCilium is Aurora’s primary Container Network Interface (CNI) and dataplane for Kubernetes workload clusters. It leverages eBPF to deliver high-performance pod networking, fine-grained network policy enforcement, and deep traffic observability through Hubble. Cilium enables advanced networking patterns, including ClusterwideNetworkPolicy and optional BGP-based route advertisement to integrate pod and service networking with enterprise firewalls.

https://cilium.io/
FalcoSecuritySIG: SecurityFalco is used within Aurora to provide runtime security monitoring and threat detection for Kubernetes workloads. It observes system calls and Kubernetes events to detect abnormal or potentially malicious behavior, such as privilege escalation, unexpected process execution, or suspicious network activity. Falco enables real-time alerting and supports policy-driven runtime enforcement aligned with zero-trust and defense-in-depth security models.

https://falco.org/
Fluent OperatorObservabilitySIG: ObservabilityFluent Operator is used within Aurora as the control plane for Kubernetes logging. It manages the lifecycle, configuration, and rollout of Fluent Bit instances across clusters using Kubernetes custom resources. By separating log pipeline configuration from node-level agents, Fluent Operator enables consistent, declarative, and multi-tenant logging practices while reducing operational complexity for platform and application teams.

https://github.com/fluent/fluent-operator
GatekeeperSecuritySIG: SecurityGatekeeper is used within Aurora as the Kubernetes admission controller for enforcing policy and governance controls at resource creation and update time. Built on Open Policy Agent (OPA), Gatekeeper enables policy-as-code enforcement for security, compliance, and platform guardrails, ensuring that Kubernetes resources adhere to approved standards before they are admitted into the cluster. This approach reduces operational risk and supports consistent enforcement of enterprise and regulatory requirements.

https://open-policy-agent.github.io/gatekeeper/
HubbleObservabilitySIG: NetworkingHubble is the observability component of Cilium that provides deep visibility into Kubernetes network traffic using eBPF. It captures network flows at the kernel level and exposes them through APIs, CLI tools, and a web-based UI. Within Aurora, Hubble enables platform operators and application teams to understand service-to-service communication, troubleshoot connectivity issues, and validate network policy behavior without packet capture or intrusive instrumentation.

https://cilium.io/hubble/
IstioNetworkingSIG: NetworkingIstio is used within Aurora to provide service mesh capabilities for Kubernetes workloads, including traffic management, mutual TLS, identity- aware authorization, and service-to-service observability. It enables fine-grained control over east-west traffic and supports zero-trust networking patterns by enforcing strong identity and policy at the application layer, complementing lower-level network controls provided by the CNI.

https://istio.io/
kube-benchPlatform SecuritySIG: Securitykube-bench is used by Aurora to assess Kubernetes clusters against the Center for Internet Security (CIS) Kubernetes Benchmarks. It evaluates cluster configuration, control plane components, and node-level settings to identify deviations from recommended security baselines. kube-bench supports compliance reporting, security posture assessments, and continuous improvement of cluster hardening across Aurora-managed environments.

https://github.com/aquasecurity/kube-bench
LokiPlatform ObservabilitySIG: ObservabilityLoki is used by Aurora as the centralized log aggregation and storage system for platform and application logs generated across Kubernetes clusters. Designed to be cost-effective and tightly integrated with Kubernetes, Loki indexes metadata rather than full log contents, enabling efficient querying and correlation of logs with metrics and traces. Loki supports operational troubleshooting, security investigations, and audit use cases across the Aurora platform.

https://grafana.com/oss/loki/
OpenCostPlatform ManagementSIG: ArchitectureOpenCost is used within Aurora to provide transparent, real-time visibility into the cost of running Kubernetes workloads. It allocates infrastructure and cloud resource costs to namespaces, workloads, and labels, enabling platform operators and client teams to understand consumption patterns, support chargeback or showback models, and make informed decisions about resource usage. OpenCost promotes financial accountability while remaining vendor-neutral and aligned with open-source cost management practices.

https://www.opencost.io/
PodTrackerPlatform ObservabilitySIG: ArchitecturePodTracker is a custom Aurora platform component that observes Kubernetes pod lifecycle events and records authoritative metadata about running workloads. It provides a reliable source of truth for pod identity, ownership, namespace context, and runtime attributes. PodTracker enables downstream automation and integration use cases, including networking, security, observability, and audit workflows, by making pod-level metadata consistently available outside the cluster.

https://github.com/gccloudone-aurora/podtracker
PrometheusObservabilitySIG: ObservabilityPrometheus is used within Aurora as the primary metrics collection and querying system for Kubernetes clusters and platform services. It scrapes metrics from workloads, infrastructure components, and platform services, storing time-series data that enables monitoring, alerting, and performance analysis. Prometheus forms the foundation of Aurora’s observability stack and supports standardized metrics-based visibility across multi-tenant environments.

https://prometheus.io/
TetragonSecuritySIG: SecurityTetragon is used within Aurora to provide real-time runtime security enforcement and deep system observability for Kubernetes workloads using eBPF. It enables policy-driven detection and prevention of undesired process, file, and network activity at the kernel level, without requiring application instrumentation. Tetragon complements admission-time controls and passive detection by enabling active enforcement and fine-grained runtime policy aligned with zero-trust and defense-in-depth security models.

https://tetragon.io/
Trivy OperatorPlatform SecuritySIG: SecurityTrivy Operator is used by Aurora to perform continuous, automated security scanning of Kubernetes workloads, container images, and cluster configurations. It integrates Trivy’s vulnerability, misconfiguration, and compliance scanning capabilities directly into the Kubernetes control plane by producing Kubernetes-native custom resources. Trivy Operator enables proactive risk identification, supports supply chain security objectives, and provides visibility into security posture across Aurora-managed clusters.

https://aquasecurity.github.io/trivy-operator/
Trust ManagerPlatform SecuritySIG: SecurityTrust Manager is used by Aurora to centrally manage and distribute trusted certificate authority (CA) bundles across Kubernetes clusters and workloads. It enables consistent trust configuration for platform and application components by synchronizing CA certificates into Kubernetes-native resources. Trust Manager reduces configuration drift, simplifies certificate trust management, and supports secure service-to-service communication in zero-trust and enterprise environments.

https://cert-manager.io/docs/projects/trust-manager/
VeleroPlatform ResilienceSIG: ArchitectureVelero is used by Aurora to provide backup and restore capabilities for Kubernetes clusters and their associated resources. It enables disaster recovery, cluster migration, and operational resilience by capturing the state of Kubernetes objects and, where supported, persistent volume data. Velero supports scheduled and on-demand backups and integrates with cloud object storage to retain backups outside the cluster lifecycle.

https://velero.io/
